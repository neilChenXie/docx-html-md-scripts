# html_to_md_v2 及与 html_to_markdown 差异分析

**日期**：2026-05-13
**文件**：`new/html_to_md_v2.py` vs `src/docx_md_tools/html_to_markdown.py`

---

## 1. Word/MSO 自动编号标题处理（核心新增功能）

这是 v2 最关键的改进。Word 导出的 HTML 中，自动编号标题会包含 `mso-list` 样式和条件注释包裹的编号 span，v1 **完全忽略**了这个问题，直接用 `get_text()` 提取标题文本，导致：

- 自动编号（如 "1.1"、"第一章"）可能被丢弃或重复
- 条件注释 `<!--[if supportLists]-->...<!--[endif]-->` 中的 span 可能被当作正文文本

**v2 的处理方式：**

- 新增 `_get_heading_text()` 函数（110-193行），专门处理含 `mso-list` 的标题：
  - 识别条件注释块 `<!--[if supportLists]-->`，将其内部标记为 `in_conditional`
  - 对 `mso-list:Ignore` 的 span（编号标签），提取其文本内容并保留编号文本（如 "1.1"）
  - 对 `<a>` 标签（书签锚点）直接跳过
  - 对 `<b>`、`<span>` 等保留其文本内容
- 新增 `_clean_word_markup()` 函数（256-283行），在解析前**预处理**整个文档：
  - 删除 `<o:p>` 标签（Word 特有段落标签）
  - 对 `supportFields` 条件注释调用 `_remove_conditional_block()` 整块移除
  - 对 `supportLists` 条件注释：
    - 若在标题内，调用 `_remove_heading_list_block()`，保留编号文本、移除锚点和忽略 span
    - 若在非标题元素中，整块移除
  - 移除所有 ProcessingInstruction

## 2. 条件注释清理（显著增强）

| 功能 | v1 | v2 |
|---|---|---|
| `[if !supportLists]` | 后处理正则移除 | 预处理结构化移除 + 后处理正则兜底 |
| `[if supportFields]` | ❌ 不处理 | ✅ 预处理整块移除（含 PAGEREF 等） |
| `[endif]` | 后处理正则移除 | 预处理 + 后处理正则 |
| `[if xxx]` 其他条件注释 | ❌ 不处理 | ✅ 后处理正则 `\[if\s+[^\]]*\]` 兜底 |
| ProcessingInstruction | ❌ 不处理 | ✅ 预处理移除 |

v1 仅在后处理阶段用简单正则清理，无法处理条件注释包裹的**HTML结构**（如 `<a>`、`<span>` 等嵌套标签）。v2 先在 BeautifulSoup 层面结构化移除整块内容，再用正则兜底清理残留。

---

## 3. 后处理 `_post_process()` 大幅增强

v1 的后处理（209-215行）仅 6 行正则替换。v2 的 `_post_process()`（370-410行）增加了：

| 新增清理项 | 说明 |
|---|---|
| `PAGEREF xxx \h` | Word 书签引用残留 |
| `mso-element:field-*` | Word 字域标记 |
| `font-family:...` | 字体声明残留 |
| `mso-spacerun:'yes'` | MSO 空格运行标记 |
| `mso-fareast-font-family:...` | 远东字体声明 |
| `mso-ansi-font-weight:...` | ANSI 字重声明 |
| `mso-bidi-font-weight:...` | Bidi 字重声明 |
| `font-size:X.Xpt` | 字号声明 |
| `font-variant/style:...` | 字体变体/风格 |
| `tab-stops:...` | 制表位声明 |
| `mso-list:...` | 列表样式声明 |
| `<span>...</span>` span | 空标签清理（多步） |
| `<a name="..."></a>` | 书签锚点清理 |
| `<o:p></o:p>` | Word 段落标签残留 |
| 残留HTML标签 | `</?[a-zA-Z][^>]*>` 最终兜底 |

v2 对 Word HTML 的 CSS/HTML 残留清理远比 v1 彻底，v1 的输出可能仍包含大量 `font-family:`、`mso-*` 等样式文本。

## 4. 编码处理改进

| 项目 | v1 | v2 |
|---|---|---|
| 检测到 GB2312 | 直接使用 | ✅ 升级为 GB18030（兼容性更好，GB18030 是 GB2312/GBK 的超集） |
| 编码优先列表 | `[detected, utf-8, gb2312, gbk, gb18030]` | `[detected(upgraded), utf-8, gb18030, gbk, gb2312]` |
| 编码使用反馈 | 仅打印 `Using encoding: {enc}` | ✅ 记录 `used_enc`，debug 模式下对比检测/实际编码 |

---

## 5. Debug 诊断模式（全新功能）

v2 新增 `--debug` 命令行参数（argparse），提供：

- `_debug_report()`（340-367行）：转换前报告标题数量、mso-list vs plain 统计、图片信息、残留注释统计
- `_get_heading_text()` 内的逐子元素调试输出
- 编码/文件大小信息
- 图片引用计数（convert_element 输出 vs 后处理输出对比，追踪图片是否被意外删除）

---

## 6. CLI 接口改进

| 项目 | v1 | v2 |
|---|---|---|
| 参数解析 | 手动 `sys.argv` | ✅ `argparse`（更规范，自带帮助） |
| 返回值 | `md_path` 或 `None` | `True` 或 `False`（布尔语义更清晰） |
| 错误信息语言 | 英文 | 中文 |

---

## 7. 其他细微差异

- v2 导入了 `Comment` 和 `ProcessingInstruction`，用于结构化识别和处理 HTML 注释/指令节点
- v2 的 `convert_element()` 传递 `debug` 参数到递归调用中
- `_td_content()` 和 `convert_table()` / `convert_list()` 两版完全相同，无变化

---

## 总结

v2 的核心价值是针对 Word/WPS 导出 HTML 的三大痛点做了结构化处理：

1. **自动编号标题** — 不再丢失或重复编号
2. **条件注释** — 结构化移除而非简单正则替换，避免残留 HTML 标签和 PAGEREF 等
3. **Word 样式残留** — 大量 MSO/CSS 属性的正则清理，输出更干净的 Markdown

对于简单的 Pandoc HTML 或标准 HTML5，两者输出差异不大。对于含自动编号标题的中文 Word/WPS 文档，v2 是必要的升级。
