## 详细变更

### 1. `src/docx_md_tools/py_docx_to_html.py`（新增，302 行）

Pandoc 跨平台 docx → HTML 转换器，整合 0501-0505 分支编号处理能力，同时保留 Windows COM 回退。

**所含函数：**

| 函数 | 行数 | 作用 | 来源 |
|------|------|------|------|
| `_extract_numbered_headings(docx_path)` | 20–89 | 解析 docx XML（`document.xml` + `numbering.xml`），提取自动编号标题文本及编号字符串 | 0501 分支核心代码 |
| `_inject_numbered_headings(html_path, numbered_headings)` | 92–139 | 在 Pandoc 生成的 HTML 中注入 `<span class="header-section-number">` 编号前缀 | 0501 分支核心代码 |
| `py_docx_to_html(input_file, output_file="")` | 142–214 | Pandoc 路线主转换函数：`.docx` → `.html`，含编号注入后处理 | 新 API |
| `main()` | 217–230 | Pandoc 路线 CLI 入口（对应 `docx2html` 命令） | 重构旧入口 |
| `docx_to_html_win(input_file, output_file)` | 233–274 | Windows COM 回退（调用 `scripts/docx_to_html.ps1`） | 新增 API |
| `win_main()` | 277–298 | Windows COM 路线 CLI 入口（对应 `docx2html-win` 命令） | 新增入口 |

**编号提取的逻辑要点**（`_extract_numbered_headings`）：
- 解析 `word/numbering.xml` 获取 `numId → abstractNumId → lvlText` 映射链
- 遍历 `word/document.xml` 中所有 `<w:numPr>` 段落
- 维护每个 `numId` 各 `ilvl` 的嵌套计数器（更深层级时重置子级）
- 用 `lvlText` 模板（如 `%1.%2`）填充编号字符串
- 未出现的父级用该级的 `start` 值填充（而非 0）
- 过滤掉纯数字编号（如 `1)` 的列表项）

**标题注入的逻辑要点**（`_inject_numbered_headings`）：
- 在 HTML 中匹配 `<p><strong>文字</strong></p>`（Pandoc 生成的自动编号段落格式）
- 替换为 `<hn><span class="header-section-number">编号</span> 文字</hn>`
- 层级映射：`ilvl → h2`（ilvl=1→h2, ilvl=2→h3, …）
- 处理嵌套列表中的标题提升（`li/section/ul/ol` 中向上提升）

### 2. `src/docx_md_tools/py_html_to_markdown.py`（新增，229 行）

Python-only HTML → Markdown 转换器，专为路线 2（Pandoc HTML 路径）设计，输出格式匹配参考答案。

**与 `html_to_markdown.py` 的主要差异：**

| 维度 | `html_to_markdown.py`（COM 路线） | `py_html_to_markdown.py`（Pandoc 路线） |
|------|----------------------------------|----------------------------------------|
| 编号标题 | 无，由 WPS 直接生成 | 解析 `<span class="header-section-number">` 并注入 `编号 **标题**` |
| 无序列表 | `- ` 前缀（标准 MD） | `Ø` 前缀（匹配参考答案） |
| 有序列表 | `1. ` 前缀 | `1)` 前缀（匹配参考答案） |
| 列表项分隔 | 单换行 | 双换行（`\n\n`） |
| 表格列表项 | `_td_content()` 中用空格连接 | `_td_content()` 中用 `Ø` 分隔 |
| 图片 | 支持 `<img>` 转换 | 不支持 |
| 返回值 | `md_path` 或 `None` | `True` 或 `False` |

**所含函数：**

| 函数 | 行数 | 作用 |
|------|------|------|
| `_td_content(td)` | 14–21 | 表格单元格内容提取，含列表项 `Ø` 分隔符 |
| `_convert_table(table)` | 24–75 | 带 colspan/rowspan 的表格 → GFM 表格 |
| `_convert_list(lst, ordered, level)` | 78–90 | 列表转换，`Ø`/`n)` 前缀格式 |
| `_convert_element(elem)` | 93–148 | 递归 HTML 元素转换（含 numbered heading、span 处理） |
| `py_html_to_markdown(html_path, md_path)` | 151–211 | 主转换函数，含多编码尝试 |
| `main()` | 214–225 | CLI 入口 |

## 架构设计

### 模块命名约定

| 前缀 | 含义 | 文件 |
|------|------|------|
| `py_` | Python 纯 Python 实现，跨平台 | `py_docx_to_html.py`、`py_html_to_markdown.py` |
| 无前缀 | 既有模块，集成了 COM 路线能力 | `html_to_markdown.py`、`md_to_docx.py` |

### 两条技术路线完整链路

```
路线 1（Windows COM，高保真）：
  WPS/Word COM → [docx_to_html.ps1] → WPS HTML → [html_to_markdown.py] → Markdown
  命令行：docx2html-win in.docx → html2md.html → .md

路线 2（Pandoc 跨平台，含编号）：
  Pandoc → HTML5 → [编号标题注入] → Pandoc HTML → [py_html_to_markdown.py] → Markdown
  命令行：docx2html in.docx → html2md.html → .md
  注：两个 html2md 共用一个命令名，默认指向路线 2；路线 1 可手动调用 html_to_markdown()
```

### 公共 API 总览

```python
# 路线 1（Windows，高保真）
docx_to_html_win(input_docx, output_html=None) -> str  # 抛出 FileNotFoundError, RuntimeError

# 路线 2（跨平台 Pandoc）
py_docx_to_html(input_docx, output_html="") -> str       # 抛出 FileNotFoundError, ValueError

# HTML → MD
html_to_markdown(html_path, md_path=None) -> str | None   # 路线 1 格式
py_html_to_markdown(html_path, md_path=None) -> bool      # 路线 2 格式（参考答案匹配）

# MD → DOCX
markdown_to_docx(md_path, docx_path=None) -> bool
```

## 影响与注意事项

### Breaking Changes
- `from docx_md_tools import docx_to_html` 不再有效，需改为 `from docx_md_tools import py_docx_to_html`
- CLI `html2md` 命令从 `html_to_markdown:main` 切换为 `py_html_to_markdown:main`，输出格式发生变化（Ø 前缀 vs - 前缀）
- `html_to_markdown` 的 `main()` 仍可通过直接运行模块文件使用

### 兼容性
- `html_to_markdown.py` 完整保留，可通过 `python src/docx_md_tools/html_to_markdown.py` 或 `from docx_md_tools import html_to_markdown` 使用
- `docx_to_html_win()` 提供与旧 PowerShell 脚本等效的 Python API
- `py_html_to_markdown()` 返回 `bool`（与早期 `html_to_markdown` 一致），而 `html_to_markdown()` 返回 `str | None`

### 注意事项
- Pandoc 需单独安装（`pypandoc` 会调用系统 pandoc 可执行文件）
- `docx2html-win` 仅 Windows 下可用，依赖已安装 WPS/Word/LibreOffice
- 两条路线有各自的 HTML→MD 转换器，输出格式不兼容，不可混用
- 表格单元格内图片仅 `html_to_markdown`（路线 1）支持，`py_html_to_markdown`（路线 2）不支持
